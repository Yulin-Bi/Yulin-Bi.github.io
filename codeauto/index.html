CodeAuto,
A Lightweight Coding Agent Built in Java

Most AI coding tools are built with TypeScript or Python, creating a high barrier for Java developers. CodeAuto draws from Claude Code's design and uses pure Java 21 to deliver a lightweight, extensible AI coding agent runtime — CLI + TUI, tool calling, MCP, permission control, session management, all running locally in your terminal.

Technical Highlights

Five Pillars of the Agent Runtime

The core systems that make CodeAuto a capable, safe, and extensible coding agent -- Agent Loop, TodoList, MCP/Skills, Multi-Layer Memory, and Safety Net.

AgentLoop.runTurn() ==================== for step 0..maxSteps: | +-- microcompact() | +-- auto compact? | |-> compactWithStats() | |-> save artifact .md | +-- model.next() | |-> API call (SSE stream) | +-- final? return messages | +-- tool calls? |-> execute each tool |-> result backfill +-> loop ==================== cancel() at any checkpoint

Agent Loop -- The Execution Engine

Multi-step tool calling with automatic context management

The Agent Loop drives every conversation turn: sending messages to the model, parsing tool calls, executing them, and feeding results back -- up to 128 steps per turn. Streaming SSE deltas update the TUI in real time. Three-layer token management keeps context under control.

  • Streaming AssistantMessage via SSE text_delta, TUI renders in-place without flicker
  • Micro-compact clears old tool results at 50% context utilization, keeps last 3
  • Full compact at 80%: summarizes middle messages, saves artifact to .codeauto/compacted/
  • Cancellable at any step -- cancel() flag checked before API calls and tool execution
  • JLine Signal.INT handler ensures Ctrl+C works even when raw mode signal suppression fails

TodoList -- Multi-Step Task Tracking

MCP + Skills -- Open Extension System

Connect any tool, embed any domain knowledge

MCP brings external tools into the agent: connect databases, APIs, or custom servers via stdio or HTTP. Skills embed domain-specific instructions that persist through the entire session via system prompt injection.

  • Dual MCP transport: stdio (Content-Length / newline-json auto-negotiation) + Streamable HTTP
  • MCP tools auto-discovered and registered as native ToolDefinitions in ToolRegistry
  • Skills discovered from .codeauto/skills + .claude/skills with 3-tier resolution
  • load_skill -> SessionSkills in-memory registry -> injected into every subsequent system prompt
  • Skill index shows name + description in system prompt, reducing unnecessary list_skills calls

Multi-Layer Memory & Dynamic Context Compression

From persistent cross-session memory to real-time token management

Four layers of memory work together: L1 injects CLAUDE.md instructions + user profile + loaded skills into every system prompt. L2 persists AI-driven memories to multi-level CLAUDE.md files with ACE Bullet structured experience. L3 compresses long conversations with full artifact persistence. L4 micro-compacts stale tool results at 50% utilization.

  • InstructionLoader: 4-level CLAUDE.md chain (user > codeauto > project > project-local), later wins
  • MemoryManager: user profile at ~/.codeauto/memory/user-profile.md, project facts written to multi-level CLAUDE.md
  • AI-driven save: system prompt guides AI to save memories via save_memory tool, check contradictions before saving
  • CompactService: saves full pre-compaction context to .codeauto/compacted/compact-*.md with pointer in summary
  • Token budget: 200K context window, local estimation as fallback, provider usage priority

Safety Net -- Undo, Checkpoints & Background Tasks

Confidently let AI modify your codebase with a full safety net

Every AI action is reversible: file-level undo tracks every modification, Git checkpoints are created before each turn, and background tasks let AI manage long-running services with health monitoring.

  • Undo system tracks every file write/edit/patch, supporting single-file undo, batch undo_all, and history via undo_list
  • Git checkpoints auto-created before each turn (best-effort); checkpoint_list + checkpoint_restore for full repo rollback
  • Background task management: spawn processes, health-check via HTTP URL or TCP port, restart with validation
  • Background tasks persisted to ~/.codeauto/managed-apps.json, survive session restarts
  • TUI footer shows running background task count; /undo and /checkpoint slash commands for quick access

Get Started

Running in Seconds

Java 21 + Maven is all you need. Start with offline mock mode (no API key), or bring your own model. Add bin/ to PATH to run from anywhere.

Install & Run (Mock Mode)
# Clone and run tests
git clone https://github.com/Yulin-Bi/CodeAuto.git
cd CodeAuto
mvn test

# Start TUI -- no API key needed
mvn exec:java "-Dexec.args=--mock --tui"

# Build standalone JAR and run
mvn package -DskipTests
java -jar target/codeauto-0.1.0-SNAPSHOT-shaded.jar --tui
